The admin dashboard is no longer showing the password login form — 
it's going straight to the dashboard without asking for authentication.

Please diagnose and fix the admin auth flow. The issue is likely one 
of these:

1. The frontend is checking localStorage for an auth token and finding 
   a stale/old token from a previous session, so it thinks the user is 
   already logged in and skips the login form entirely.

2. The auth check condition got broken during a recent edit — e.g. 
   the condition was accidentally changed from "if token exists, show 
   dashboard" to always showing the dashboard.

3. The admin route guard component is not correctly checking 
   authentication state on mount.

Fix it by doing ALL of these:

FIX 1 — Clear any stale localStorage tokens:
In the admin page component, on mount (useEffect), verify the stored 
token against the backend by calling GET /api/admin/verify (create 
this endpoint if it doesn't exist). If the backend returns 401, clear 
the localStorage token and show the login form. Never trust a 
localStorage token without server-side verification on page load.

FIX 2 — Add GET /api/admin/verify endpoint:
Create a simple endpoint that reads the Authorization header, validates 
the token against the server-side session store (same way other admin 
routes do), and returns 200 if valid or 401 if not. This lets the 
frontend verify tokens on page load.

FIX 3 — Ensure the login form shows when not authenticated:
The admin page component should have clear logic:
- isAuthenticated = false by default
- On mount: call /api/admin/verify with stored token
- If 200: set isAuthenticated = true, show dashboard
- If 401 or no token: set isAuthenticated = false, show login form
- Never show dashboard content when isAuthenticated is false

FIX 4 — Clear localStorage on logout:
Confirm the Sign Out button removes the token from localStorage AND 
invalidates it on the server side.

After fixing, verify:
1. Open /admin in an incognito window — login form should appear
2. Enter wrong password — "Invalid administrative credentials" shown
3. Enter correct password — dashboard opens
4. Click Sign Out — returns to login form
5. Refresh the page after login — should stay logged in (token still valid)
6. Call GET /api/admin/stats without auth header — still returns 401
